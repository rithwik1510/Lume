## Findings considered and rejected

- **"Autorun listener dropped after timeout" (orchestrator.ts:267-281)** —
  by design: the 20s reap is documented ("give up rather than type blind");
  not a bug.
- **"Final bytes lost when Channel send fails" (pty.rs flusher)** — when
  `send` errors the channel is gone; nothing could be delivered. The real,
  smaller issue (reader not told to stop) is folded into Plan 002.
- **"attentionTracker re-entrancy / `!` assertions"** — bounded Zustand
  nested-set behavior and a safe has()/get() idiom; the tracker was already
  hardened post-2026-06-09 (throttle, session cache, sustain gate).
- **Modal micro-store factory (settingsModal/shortcutsModal/splitMenu/
  contextMenu)** — stores are semantically different (anchors, toggle,
  payloads); a factory would save ~40 LOC and add indirection. Not worth doing.
- **Pruning @fontsource packages** — all 6 are live via `fontPairs.ts`
  font-pair presets; not dead weight.
- **xterm registry "races"** — JS is single-threaded and the registry guards
  existence on each access; no reachable corruption found.
